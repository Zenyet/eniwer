## Overview

Plugins can contribute settings sections via the `SettingsContributor` interface. This lets plugins own their configuration UI within the settings panel.

## Sub-page Settings (Default)

By default, a plugin's settings are shown on a **dedicated sub-page**, accessible via the plugin management panel.

```typescript
import type { Plugin, SettingsContributor, PluginContext } from './plugins/types';
import type { MenuConfig } from './types';

class TimerPlugin implements Plugin, SettingsContributor {
  readonly id = 'timer';
  readonly name = 'Pomodoro Timer';
  readonly description = 'plugin.timer.description';
  private ctx!: PluginContext;

  activate(ctx: PluginContext): void {
    this.ctx = ctx;
    ctx.onConfigChange((config) => {
      // React to config changes
      const duration = config.timerDuration ?? 25;
      this.updateDuration(duration);
    });
  }

  deactivate(): void {}

  // SettingsContributor
  getSettingsHTML(config: MenuConfig): string {
    const duration = (config as any).timerDuration ?? 25;
    return `
      <div class="glass-settings-section">
        <div class="glass-settings-section-title">Pomodoro Timer</div>
        <div class="glass-form-group">
          <label class="glass-form-label">Duration (minutes)</label>
          <select class="glass-select" id="timer-duration">
            <option value="15" ${duration === 15 ? 'selected' : ''}>15</option>
            <option value="25" ${duration === 25 ? 'selected' : ''}>25</option>
            <option value="45" ${duration === 45 ? 'selected' : ''}>45</option>
          </select>
        </div>
      </div>
    `;
  }

  bindSettingsEvents(
    shadowRoot: ShadowRoot,
    tempConfig: MenuConfig,
    onChange: () => void
  ): void {
    const select = shadowRoot.querySelector('#timer-duration') as HTMLSelectElement;
    select?.addEventListener('change', () => {
      (tempConfig as any).timerDuration = parseInt(select.value, 10);
      onChange();
    });
  }

  private updateDuration(minutes: number): void {
    // Apply new duration...
  }
}
```

## Inline Settings

Set `settingsOrder` to render settings **inline on the main settings page** instead of a sub-page. Lower values appear higher:

```typescript
class AccountPlugin implements Plugin, SettingsContributor {
  readonly id = 'account';
  readonly name = 'Account';
  readonly settingsOrder = -100; // Appears at the very top

  getSettingsHTML(config: MenuConfig): string {
    return `
      <div class="glass-settings-section">
        <div class="glass-settings-section-title">Account</div>
        <div class="glass-form-group">
          <label class="glass-form-label">Username</label>
          <input class="glass-input-field" id="username" value="${config.username || ''}">
        </div>
      </div>
    `;
  }

  bindSettingsEvents(shadowRoot: ShadowRoot, tempConfig: MenuConfig, onChange: () => void): void {
    const input = shadowRoot.querySelector('#username') as HTMLInputElement;
    input?.addEventListener('input', () => {
      (tempConfig as any).username = input.value;
      onChange();
    });
  }

  // ...
}
```

The built-in `CloudSyncPlugin` uses `settingsOrder = -100` to render the Google account / sync section at the top of the main settings page.

## CSS Classes

Use the built-in glass CSS classes for consistent styling:

| Class | Description |
|---|---|
| `glass-settings-section` | Section wrapper |
| `glass-settings-section-title` | Section heading |
| `glass-form-group` | Form field container |
| `glass-form-group glass-form-toggle` | Toggle switch layout |
| `glass-form-label` | Field label |
| `glass-select` | Styled select dropdown |
| `glass-input-field` | Styled text input |
| `glass-toggle` | Toggle switch wrapper |
| `glass-toggle-slider` | Toggle switch track |
| `glass-form-hint` | Subtle hint text below a field |
| `glass-btn` | Base button class |
| `glass-btn-primary` | Primary action button |
| `glass-btn-secondary` | Secondary action button |
| `glass-btn-danger` | Destructive action button |
| `glass-sync-chips` | Chip group container |
| `glass-sync-chip` | Individual selectable chip |

## How It Works

1. When the settings panel renders, `getSettingsHTML(config)` is called for each enabled `SettingsContributor`
2. For **inline** contributors (with `settingsOrder`): HTML is inserted at the top of the main settings page, sorted ascending
3. For **sub-page** contributors (without `settingsOrder`): HTML is rendered on a dedicated page accessible from the plugin management panel
4. `bindSettingsEvents(shadowRoot, tempConfig, onChange)` is called to wire up event handlers
5. When the user changes a value, mutate `tempConfig` and call `onChange()` to enable the Save button
6. On Save, the full config (including plugin mutations) is persisted to Chrome storage
7. The `onConfigChange` callback fires with the new config across all plugins
