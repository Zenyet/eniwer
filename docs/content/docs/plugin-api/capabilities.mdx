---
title: Capabilities
description: Mix-in interfaces for command, view, settings, and minimize support
---

Plugins gain capabilities by implementing additional interfaces alongside `Plugin`.

## CommandContributor

Contribute commands to the command palette:

```typescript
interface CommandContributor {
  /** Return the list of commands this plugin provides */
  getCommands(): MenuItem[];
  /** Handle a command action. Return true if handled. */
  handleCommand(action: string, selectedText: string): boolean;
}
```

Commands returned by `getCommands()` are merged into the palette. When the user selects a command, `PluginManager` routes it to the first plugin whose `handleCommand()` returns `true`.

If a plugin owns menu-item IDs beyond its own `id`, declare them via `menuItemIds` on the Plugin interface so they are properly hidden when the plugin is disabled.

### Example

```typescript
class MyPlugin implements Plugin, CommandContributor {
  readonly id = 'myPlugin';
  readonly menuItemIds = ['myAction', 'myOtherAction'];

  getCommands(): MenuItem[] {
    return [{
      id: 'my-action',
      action: 'myAction',
      label: 'My Action',
      icon: '<svg>...</svg>',
      enabled: true,
      order: 100,
    }];
  }

  handleCommand(action: string, selectedText: string): boolean {
    if (action !== 'myAction') return false;
    // do something with selectedText
    return true;
  }
}
```

## ViewContributor

Own a custom view inside the command palette:

```typescript
interface ViewContributor {
  /** The view type string this plugin owns */
  readonly viewType: string;
  /** Return HTML for the view */
  getViewHTML(): string;
  /** Bind DOM events after the view HTML is injected */
  bindEvents(shadowRoot: ShadowRoot): void;
  /** Called when navigating away from this view (optional cleanup) */
  onViewLeave?(): void;
}
```

Navigate to a plugin view via:

```typescript
ctx.ui.navigateToView('myViewType');
// or push onto the stack (enables back navigation)
ctx.ui.pushView({ type: 'myViewType', title: 'My View' });
```

## SettingsContributor

Contribute a settings section in the extension settings panel:

```typescript
interface SettingsContributor {
  /** Return HTML for the settings section */
  getSettingsHTML(config: MenuConfig): string;
  /** Bind settings DOM events. onChange signals that tempConfig was mutated. */
  bindSettingsEvents(
    shadowRoot: ShadowRoot,
    tempConfig: MenuConfig,
    onChange: () => void
  ): void;
  /**
   * When set, settings are rendered inline on the main settings page
   * (sorted ascending). Without this, settings appear on a sub-page.
   */
  settingsOrder?: number;
}
```

### Sub-page vs Inline Settings

By default, a `SettingsContributor`'s HTML is rendered on a **dedicated sub-page** (accessed via the plugin management panel → click on a plugin with settings).

If `settingsOrder` is set, the settings HTML is rendered **inline on the main settings page**, sorted by ascending order alongside other inline contributors. This is useful for settings that should always be visible (e.g. the Cloud Sync account section uses `settingsOrder = -100` to appear at the top).

## MinimizableContributor

Support minimize-to-background while streaming:

```typescript
interface MinimizedPluginData {
  pluginId: string;
  title: string;
  iconHtml?: string;
  isLoading: boolean;
  pluginData: unknown; // Plugin-specific state snapshot
}

interface MinimizableContributor {
  saveAsMinimized(): MinimizedPluginData | null;
  restoreFromMinimized(data: MinimizedPluginData): void;
}
```

When the command palette is hidden while a plugin view is active:
1. `saveAsMinimized()` is called — plugin returns its state snapshot
2. A minimized task badge appears in the command palette
3. When the user clicks the badge, `restoreFromMinimized(data)` is called

While minimized, plugins can update the badge via `ctx.minimizedTasks.findAndUpdate()`:

```typescript
ctx.minimizedTasks.findAndUpdate(
  this.id,
  (data) => true, // predicate to find the right task
  (task) => {
    task.content = 'Updated content...';
    task.isLoading = false;
  }
);
```

## Type Guards

Use these to check capabilities at runtime:

```typescript
isCommandContributor(plugin)    // has getCommands + handleCommand
isViewContributor(plugin)       // has viewType + getViewHTML + bindEvents
isSettingsContributor(plugin)   // has getSettingsHTML + bindSettingsEvents
isMinimizableContributor(plugin) // has saveAsMinimized + restoreFromMinimized
```
